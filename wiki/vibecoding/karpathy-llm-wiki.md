# 📓 Andrej Karpathy: LLM-Wiki Pattern

## Summary
Karpathy proposes a pattern for personal knowledge management where an LLM maintains a structured markdown wiki. This system is designed to grow more valuable over time through compounding links and structured ingestion.

## Core Architecture
- **Raw Materials (`raw/`):** Primary sources (PDFs, URLs, transcripts).
- **Structured Knowledge (`wiki/`):** Interlinked markdown files generated by the LLM.
- **Operating Manual (`GEMINI.md`):** Instructions that define formatting, schemas, and workflows.
- **Navigation Tools:** `index.md` (Map) and `log.md` (Chronology).

## Key Principles
- **Compounding Artifact:** The wiki is "code" that the LLM manages.
- **Context > RAG:** For personal scales, keeping the wiki in-context is superior to vector search.
- **Discipline:** The LLM must follow strict schemas to ensure the wiki remains a coherent "operating system" for thoughts.
